# 서귀포의료원 시설관리팀 업무공유 시스템

## 구성
- `index.html` — 시작 페이지 (월간일정 / 업무 선택)
- `board.html` — 월간일정 대시보드 (사무실 대형화면 전자칠판, 168×95cm 터치스크린)
- `input.html` — PC/모바일용 입력 화면 (대시보드 / 업무관리 / 일정·공사등록 / 전화·기타메모)
- `firebase-config.js`, `auth-gate.js`, `common.css` — 공용 모듈 (직접 열 필요 없음)

---

## 1단계. Firebase 콘솔 설정 (딱 2가지만 하면 됩니다)

### ① 익명 로그인 켜기
1. https://console.firebase.google.com → `smc-fm` 프로젝트 선택
2. 왼쪽 메뉴 **Authentication** → **Sign-in method** 탭
3. **Anonymous(익명)** 항목 클릭 → 사용 설정 → 저장

이게 안 켜져 있으면 접근코드를 입력해도 로그인이 안 되고 화면이 멈춘 것처럼 보입니다.

### ② Firestore 보안규칙 붙여넣기
1. 왼쪽 메뉴 **Firestore Database** → **규칙(Rules)** 탭
2. 아래 내용을 전체 붙여넣고 **게시(Publish)**

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

> 참고: 이 시스템은 "접근코드(3813) → 익명 로그인" 방식의 최소 인증입니다.
> 개인별 아이디/비밀번호는 없고, 코드를 아는 사람이면 누구나 접속·입력할 수 있습니다.
> (팀 내부용 도구이므로 이 정도 보안 수준으로 설계했습니다)

---

## 2단계. GitHub Pages로 올리기

1. GitHub에서 새 저장소(Repository) 생성 (예: `smc-fm`), Public 또는 Private 무관
2. 이 폴더 안의 파일 6개를 저장소에 업로드 (드래그 앤 드롭으로 가능)
   - `index.html`, `board.html`, `input.html`, `firebase-config.js`, `auth-gate.js`, `common.css`
3. 저장소 **Settings → Pages**
4. **Source**: `Deploy from a branch` / **Branch**: `main`, 폴더 `/(root)` 선택 → **Save**
5. 1~2분 후 상단에 `https://아이디.github.io/smc-fm/` 형태의 주소가 생깁니다

---

## 3단계. 사용 방법

### 월간일정 (사무실 대형화면 전자칠판)
1. 화면에서 브라우저로 `https://아이디.github.io/smc-fm/board.html` 접속
2. 접근코드 `3813` 입력 (한 번만, 세션 유지됨)
3. 이후 브라우저 탭을 항상 열어두면 됩니다. 화면보호기/절전모드로 꺼지지 않도록 안드로이드 디스플레이 설정에서 "화면 항상 켜짐"으로 맞춰두시는 걸 추천드립니다. 상단 시계에 초 단위까지 표시되어 화면이 멈추지 않고 살아있는지 한눈에 확인할 수 있습니다.
4. 좌측: 이번 달 캘린더 (공사=주황, 미팅=파랑, 업무=초록). 주말과 공휴일은 날짜 숫자가 빨간색으로 표시되고, 오늘 날짜는 보라색 원으로 구분됩니다. 날짜를 터치하면 그날 일정이 큰 글씨로 확대되어 보입니다.
5. 우측: 미확인 전화·기타메모 알림. 같은 담당자·용건이 연달아 있으면 한 줄로 묶여 "N건"으로 표시됩니다. 카드를 터치하면 이름을 선택해 (묶인 항목은 한 번에) 확인 처리할 수 있습니다. 완료된 항목은 방금 확인한 순서대로 위에 쌓입니다.
6. 24시간 넘게 미확인 상태인 카드는 은은하게 깜빡입니다.
7. 상단 시계 오른쪽의 "업무" 버튼으로 업무 화면으로 바로 이동할 수 있습니다.

### PC / 모바일 입력 (업무)
1. `https://아이디.github.io/smc-fm/input.html` 접속, 접근코드 입력
2. 접속 시 먼저 "작성자"를 선택해야 이후 화면을 사용할 수 있습니다 (혼선 방지용, 상단바 "변경" 버튼으로 언제든 다시 선택 가능)
3. 탭 4개: **대시보드 / 업무관리 / 일정·공사등록 / 전화 및 기타메모**. 상단 "월간일정" 버튼으로 전자칠판 화면으로 바로 이동할 수 있습니다.
4. 대시보드에는 현재 작성자와 관련된 업무·일정/공사·전화메모가 한 화면에 모아서 보입니다. 관련 항목에 다른 사람이 이력/상세내용을 업데이트하면 "🔔 업데이트 알림"에 뜨고, 열어보면 자동으로 사라집니다 (담당이 "전체"면 팀 전체에게, 담당자·요청자 외 다른 사람이 쓰면 둘 다에게, 둘 중 하나가 쓰면 반대쪽에게 알림). 다른 사람이 내가 작성한 항목의 삭제를 요청한 경우에도 여기서 승인/거절할 수 있습니다.
5. 업무관리에서 새 업무를 등록하면 요청자는 자동으로 현재 작성자가 되고, 담당자는 별도로 지정할 수 있습니다. 항목을 클릭하면 상태(요청됨/진행중/완료/보류)를 바꾸고 상세 이력을 보며 진행상황을 추가할 수 있습니다.
6. 일정·공사 등록 시 날짜는 "기간"과 "개별 날짜"를 함께 조합해서 추가할 수 있고, 숫자만 입력해도 자동으로 년-월-일 형식이 채워지며 팝업 달력에서 주말·공휴일(대한민국 법정 공휴일 기준)이 빨간색으로 표시됩니다. 목록에서는 연속된 날짜가 "몇일~몇일" 범위로 압축되어 표시됩니다. 시간도 선택 입력할 수 있고, 담당은 특정 팀원 외에 "전체"·"미지정"으로도 지정할 수 있습니다. **일정/공사는 작성자(등록자) 본인만 항목을 수정할 수 있고, 그 외 팀원은 "상세내용"을 이력으로 추가만 할 수 있습니다** (누가 언제 무엇을 추가했는지 이력으로 남습니다).
7. 모든 목록은 각 행 왼쪽 체크박스로 여러 건을 한 번에 선택해 상단 "선택 삭제" 버튼으로 일괄 삭제할 수 있고, 개별 행의 🗑 아이콘이나 상세화면에서도 삭제할 수 있습니다. 본인이 작성한 항목은 바로 삭제되고, 다른 사람이 작성한 항목을 삭제하려 하면 작성자에게 삭제 요청이 전달되어 작성자가 대시보드에서 동의해야 실제로 삭제됩니다.

## 데이터 보관
모든 데이터는 Firestore에 계속 누적 저장됩니다 (자동 삭제 없음). 나중에 필요하면 Firebase 콘솔의 Firestore Database 화면에서 직접 데이터를 조회·내보내기 할 수 있습니다.

## 나중에 팀원 명단을 바꾸려면
`firebase-config.js` 파일을 열어 아래 줄의 이름만 수정하면 모든 화면(전자칠판, 입력화면)에 자동 반영됩니다.

```js
export const TEAM_MEMBERS = ["강보선", "강은석", "박재현", "김준형", "김류현"];
```

## 접근코드를 바꾸려면
같은 파일에서 아래 줄을 수정하세요.

```js
export const ACCESS_CODE = "3813";
```
